**Task function `scrape_episode(episode_id)`:**
1. Load episode, set status to `scraping`
2. If `scraped_html` empty: fetch + clean + store
3. If all required fields present (user filled them): advance to `downloading`, return
4. Call LLM provider with prompt + HTML
5. Apply extracted fields (only update empty ones)
6. If all required fields now present: status ŌåÆ `downloading`
7. Else: status ŌåÆ `needs_review`
8. On any exception: status ŌåÆ `failed`

### 9. Assistant ŌĆö Fix OpenAI API Call

The `text.format` dict was passing the schema nested under a `schema` key:
```python
# Before (broken)
text={"format": {"type": "json_schema", "schema": response_schema}}

# After (fixed)
text={"format": {"type": "json_schema", **response_schema}}
```

This spreads `name`, `strict`, and `schema` from `SCRAPE_RESPONSE_SCHEMA` directly into the format dict, giving the API the required `text.format.name` parameter.
