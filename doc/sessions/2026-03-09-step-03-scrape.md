# Session: Step 3 — Scrape

**Date:** 2026-03-09

## Summary

Implemented Step 3 of the RAGtime pipeline: scraping episode metadata from podcast web pages using an LLM. This session introduced Django Q2 for async task processing, the full LLM provider abstraction, and the scraper module.

## Conversation Flow

1. **Planning phase** — Discussed requirements for the scrape step. User provided clarifications via a structured Q&A template:
   - Fields: title, description, published_at, image_url, language, audio_url
   - Store cleaned HTML for debugging
   - New `scraping` status between pending and downloading
   - Full provider abstraction, OpenAI first
   - Auto-trigger scrape on episode creation
   - `needs_review` status for incomplete extractions
   - User added requirement to also extract the MP3 audio URL

2. **Plan document** — Created `doc/plans/step-03-scrape.md` with full architecture, status transitions, implementation steps.

3. **Implementation** — Executed the 10-step plan:
   - Added dependencies (django-q2, openai, httpx, beautifulsoup4, python-dotenv)
   - Configured settings (dotenv, Q_CLUSTER, RAGTIME_LLM_* vars)
   - Extended Episode model with metadata fields and new statuses
   - Created provider abstraction package (base ABCs, OpenAI implementation, factory)
   - Implemented scraper module (HTML fetch/clean, LLM prompt, task function)
   - Added post_save signal for auto-triggering
   - Enhanced admin with conditional editing, fieldsets, reprocess action
   - Generated and applied migrations
   - Updated README.md pipeline table
   - Wrote 25 tests covering all components

4. **Bug fix** — Fixed OpenAI API call: the structured output format needed `name` at the top level of the format dict, not nested under a `schema` key. Changed from `{"schema": response_schema}` to `{**response_schema}` to spread name/strict/schema into the format object.

5. **README update** — Added `RAGTIME_LLM_MODEL` environment variable to the Configuration section.

## Key Decisions

- **OpenAI Responses API** with JSON schema for structured extraction (not Chat Completions)
- **ORM broker** for Django Q2 (no Redis dependency)
- **Smart reprocessing**: skips HTTP fetch if HTML cached, skips LLM if user already filled required fields
- **Required fields** for advancing: only `title` and `audio_url` — other fields are nice-to-have
- **HTML cleaning**: strips script/style/nav/footer tags, truncates to 30k chars, preserves meta/audio tags
