# Step 8: Chunk Transcript

**Date:** 2026-03-14

## Problem

The pipeline needed a chunking step to split Whisper transcripts into smaller segments for future embedding and entity-to-chunk mapping. Previously chunking was listed as Step 10 sharing the `embedding` status. It needed its own status and earlier placement in the pipeline (before extraction) to enable future per-chunk entity extraction.

## Changes

- **Episode status** (`episodes/models.py`): Added `CHUNKING = "chunking"` between `SUMMARIZING` and `EXTRACTING`. Updated `PIPELINE_STEPS` accordingly.

- **Chunk model** (`episodes/models.py`): New model with `episode` FK, `index` (0-based), `text`, `start_time`/`end_time` (seconds), `segment_start`/`segment_end` (Whisper segment IDs, inclusive). Unique constraint on `(episode, index)`, ordered by `index`.

- **Chunker module** (`episodes/chunker.py`): Two functions:
  - `chunk_transcript(transcript_json, target_words, overlap_segments)` — pure function that iterates Whisper segments, accumulates until word count exceeds target, then starts a new chunk with overlap from previous chunk's last segment.
  - `chunk_episode(episode_id)` — pipeline task that validates status, deletes old chunks (re-processing), calls `chunk_transcript()`, bulk-creates `Chunk` objects, advances to `EXTRACTING`.

- **Summarizer** (`episodes/summarizer.py`): Now advances to `CHUNKING` instead of `EXTRACTING`.

- **Signal handler** (`episodes/signals.py`): Added `CHUNKING` status dispatch to queue `episodes.chunker.chunk_episode`.

- **README/AGENTS.md**: Updated pipeline table to reflect new step order.

## Key Parameters

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| `CHUNK_TARGET_WORDS` | 150 | Balances chunk granularity with embedding context window |
| `CHUNK_OVERLAP_SEGMENTS` | 1 | Single segment overlap provides continuity without excessive duplication |
